# Hidden Requirement IDs in Exported Workbooks

Exported member-response workbooks should include hidden stable requirement IDs for rows
generated by CMM, because visible outline numbers can change when requirements are
inserted, deleted, or moved. Imports should still fall back to requirement number and
text matching when those hidden IDs are missing or damaged, because potential consortium
members may manually edit the spreadsheet before returning it.

**Consequences**

- CMM-authored workbooks can map responses back to requirements more reliably than
  visible numbering alone.
- The import review flow still needs mismatch handling for edited, copied, or manually
  reconstructed workbooks.
- Because the app has not launched yet, no legacy workbook compatibility path is required
  beyond the number/text fallback.
